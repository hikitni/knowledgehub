# AGENTS.md

## KnowledgeHub

当任务涉及项目背景、PRD、技术方案、业务规则、历史决策、故障记录、接口契约、部署流程或运行手册时：

1. 先调用 `kb_project_context`，传入当前项目的绝对路径，确认 `kb_id` 与 `project_id`。
2. 再调用 `kb_search`，优先限定当前 `project_id`；跨项目问题才扩大到对应 `scopes`。
3. 对支撑结论的命中调用 `kb_read`，读取原文章节，不只依赖摘要片段。
4. 回答必须标注文档标题、绝对路径、章节路径和行号范围。
5. 未检索到时明确说明，不得只依赖模型记忆编造项目事实。

职责边界：

- 代码结构、符号定义、调用链和变更影响：优先使用 CodeGraph。
- PRD、README、架构文档、ADR、业务背景和历史事实：优先使用 KnowledgeHub。
- 精确字面量、日志文本和尚未入索引的工作区变更：使用 `rg` 或直接读取指定文件。
- 外部文档原文抓取：使用已配置的企业连接器；落地到本地后由 KnowledgeHub 索引。
